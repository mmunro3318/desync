## The Problem

`unity-DESYNC/Assets/_Project/Scenes/House_Graybox.unity` has a **known floor-to-floor light leak** — light from one floor bleeds into the other. This was identified during migration but never root-caused.

## Recommended Skill: `/investigate`

We had good results with `/investigate` for the multiplayer fix. It enforces the discipline of: investigate → analyze → hypothesize → implement (no fixes without root cause). This is the right approach for a graphics issue where the root cause could be any of several URP subsystems.

**Alternative skills if `/investigate` doesn't fit:**
- `/diagnose` — similar structured debugging loop
- `/superpowers:systematic-debugging` — if the issue turns out to be multi-layered

## What to Read First

1. `docs/design/98-unity-research/03-unity-urp-graphics-lighting-horror-report.md` — our existing URP research
2. `ROADMAP.md` — S0.1 section for acceptance criteria
3. The scene itself via Unity MCP (`/unity-mcp-skill`) to inspect light objects, volume profiles, and shadow settings

## Likely Investigation Vectors

- Shadow cascade configuration (distance/splits may not cover floor separation)
- Light probe bleed between floors (probe placement or baking issue)
- Reflection probe boundaries crossing floors
- Missing or incorrect occlusion geometry between floors
- URP renderer feature settings (shadow bias, normal bias)
- Volume profile override boundaries (is the atmosphere volume global when it should be per-floor?)

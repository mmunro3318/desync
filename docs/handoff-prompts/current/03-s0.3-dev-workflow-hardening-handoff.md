# Handoff Prompt: S0.3 — Dev Workflow Hardening

## Context

Post-S0.1/S0.2, the retro identified three gaps before S1A (House Graph Authoring) begins:

1. **Zero test coverage.** The `Desync.Tests.EditMode` asmdef exists with one test (`NetworkBootstrapConsistencyTests`), but no scene-state validation tests. The light leak fix took 5 hypothesis rounds partly because there was no programmatic way to assert "geometry doesn't protrude through walls."
2. **No PR workflow.** All 14 commits went direct to main. Mike is a solo dev unfamiliar with Git branching/PR workflows. Claude Code has `/ship` and `/review` skills that can enforce a lightweight PR flow, but the scaffolding needs to be established first.
3. **Branch cleanup ambiguity.** The `debug/light-leak-fix` branch was merged to main but git reported "not fully merged" on delete attempt — likely because the merge was a fast-forward or the branch has diverged commits. Needs investigation and cleanup.

## Scope (tight — this is a mini-sprint, not a feature)

- [ ] Write a `HouseGrayboxGeometryTests` EditMode test that validates floor/ceiling rects don't protrude past exterior walls
- [ ] Establish a branch + PR workflow convention documented in CLAUDE.md
- [ ] Clean up the `debug/light-leak-fix` branch
- [ ] Verify `NetworkBootstrapConsistencyTests` still passes

## Task 1: Scene-State Validation Test

**What:** An EditMode test in `unity-DESYNC/Assets/_Project/Tests/EditMode/` that opens `House_Graybox.unity` and asserts:
- All floor/ceiling piece bounds are within exterior wall inner edges (the light leak invariant)
- GF_Ceiling top Y <= GF wall top Y (ceiling flush with walls)
- SF_Ceiling top Y <= SF wall top Y
- No coplanar faces between GF_Ceiling and SF_Floor pieces (Y positions differ by >= 0.01)
- All floor/ceiling MeshRenderers have `shadowCastingMode == TwoSided`

**Pattern to follow:** `NetworkBootstrapConsistencyTests` — opens a scene in EditMode, queries serialized state, asserts. No Play mode needed.

**Why this matters:** The next time someone touches House_Graybox geometry (or when the house graph spawns room nodes), this test catches light leak regressions in CI before anyone opens Unity.

**Recommended skill:** `/tdd` — write the failing test first, then verify it passes against the current scene state (it should pass immediately since the fix is already in).

## Task 2: Branch + PR Workflow

**What:** Establish a lightweight PR convention for this solo project:
- Feature/fix work happens on branches (`feat/`, `fix/`, `debug/` prefixes)
- PRs are created via `/ship` skill which runs tests, reviews diff, bumps version, creates PR
- PRs are reviewed via `/review` skill (AI code review)
- Merge to main via GitHub (squash merge preferred for clean history)

**Document in CLAUDE.md** under a new `## Git workflow` section:
- Branch naming convention
- When to branch (any non-doc, non-trivial change)
- PR creation flow (`/ship`)
- PR review flow (`/review`)
- Merge strategy (squash merge to main)

**Why this matters:** S1A will produce runtime C# code. Direct-to-main commits with no review gate is how bugs compound. The `/ship` + `/review` skills are already available and powerful — they just need the convention established.

**Recommended skill:** This is a docs + config task. No special skill needed — just write the CLAUDE.md section and test the `/ship` flow on a throwaway branch.

## Task 3: Branch Cleanup

**What:** Investigate and resolve the `debug/light-leak-fix` branch state:
```bash
git branch -d debug/light-leak-fix  # if this fails with "not fully merged"
git log main..debug/light-leak-fix  # check for commits not on main
git log debug/light-leak-fix..main  # check for commits not on the branch
```

If the fix commits are confirmed on main, force-delete is safe: `git branch -D debug/light-leak-fix`.

## Task 4: Verify Existing Tests

**What:** Run `NetworkBootstrapConsistencyTests` to confirm no regressions from the scene changes.

```bash
# Via Unity MCP:
run_tests testMode="EditMode" filter="NetworkBootstrapConsistencyTests"
```

## Recommended Skill Flow

1. **Start:** `/tdd` for the geometry validation test (Task 1)
2. **Then:** Write the CLAUDE.md git workflow section (Task 2) 
3. **Then:** Branch cleanup (Task 3) — manual git commands
4. **Then:** `/ship` to create a PR for the test + docs changes (validates the new workflow)
5. **End:** `/review` on the PR (validates the review flow)

This mini-sprint should take one focused session. It produces:
- A regression test that locks down the light leak fix
- A documented PR workflow ready for S1A
- A clean git state with no dangling branches

## What NOT to Do

- Don't start S1A work in this session — this is scaffolding only
- Don't write complex test infrastructure — one test file, simple assertions
- Don't set up CI/CD — that's a later concern (the test just needs to run locally via Unity MCP or the Test Runner window)
